Recommendations

W3 html validation

  • The first thing I noticed is that Bleacher Report is using a lot of obselete html code, especially with its video (iframe) content. This code should be updated by using CSS instead.
  • The second thing Bleacher Report could improve on is cleaning up what looks to be old sections of their code. Specifically, they have the element itemProp throughout their code, but it is not being used as a property of any other item. This can be cleaned up to make the page more easily accessible.

CSS validation

  • The CSS used on Bleacher has 14 errors reported. The first issue they could fix is that many of the errors are due to them having properties listedin their code that do not actually exist. Cleaning up these obselete/unused properties could help their page load times and present a cleaner code to outside viewers.
  • A second issue they should address is that many of their videos do not show still image previews while viewed on a mobile device. I am not certain if any of the errors on this CSS validation report correspond to this problem, but fixing this will give their mobile users a better, more enticing viewing experience.

PageSpeed validation

  • The PageSpeed report for Bleacher Report gives it Average and Medium marks on desktop, and Average and High marks on mobile.
  • This average speeds are given because Bleacher Report has many uncompressed videos, slowing down the average load times for all users. Compressing these files would solve the problem.
  • Another thing Bleacher Report could do to increase its optimization according to PageSpeed is to Leverage browser caching. This could be done by instructing the browser to load previously downloaded local resources instead of loading all resources over the network. This would mean that repeat users would get a faster experience at Bleacher Report.
